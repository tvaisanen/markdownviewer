# PDF Export — Design

**Date:** 2026-04-22
**Status:** Approved, pending implementation plan

## Goal

Export the currently rendered markdown document to a PDF that looks exactly
like the on-screen render, with paged-media chrome added: intelligent
section-aware pagination, never-split images and diagrams, and presentable
typography ready to hand to someone.

Browser-style "Save as PDF" is not acceptable quality. The export must feel
as premium as the viewer itself.

## Principle: WYSIWYG

The PDF is a paged version of what's on screen. The same rendering pipeline
drives both. There is no parallel print stylesheet. The PDF only *adds* a
thin paged-media overlay on top of the active screen CSS:

- `@page` size, margins, page numbering
- Break rules (avoid splits inside figures, diagrams, code, tables;
  keep headings with their following content)
- Force white background (brightness slider does not carry over to paper)

Mermaid theme, syntax highlighting theme, typography — all inherited from
the on-screen render.

## User-facing behavior

### Triggers

- **⌘P** opens the PDF Preview window and primes the native print dialog.
- **File → Export as PDF…** opens the PDF Preview window primed for the
  "Export" button (save panel).
- Both paths use the identical rendering pipeline.

### Preview window

A separate, non-modal window. Stays open while the user edits the source;
re-renders on file change via the existing `FileWatcher`.

Layout:

- **Left sidebar:** page thumbnails (`PDFThumbnailView`). Pages where
  content was scaled below 70% to fit get a small warning badge so the user
  knows to inspect them.
- **Center:** full-page `PDFView`.
- **Toolbar:**
  - Theme picker (Technical / Apple Docs / GitHub) — changes document theme
    on screen *and* in the preview (single source of truth)
  - Paper size (Letter / A4), auto-initialized from
    `NSPrintInfo.sharedPrintInfo.paperSize`, choice persisted
  - Orientation (Portrait / Landscape), applies to the whole document
  - "Start new page at H1" toggle (off by default; smart pagination
    handles it otherwise)
  - Header / footer toggles (defaults come from the active theme; user can
    override)
  - Header text field (defaults to the document filename without its
    extension)
  - **Print…** (⌘P) and **Export…** buttons

### Document themes

Three built-in themes applied document-wide (screen + PDF):

- **Technical paper** — serif body (Charter/New York), text-dense,
  arXiv-style. Default header/footer: title top-right, page number
  bottom-center.
- **Apple Documentation** — SF Pro, generous whitespace, soft gray rules.
  Default header/footer: page number bottom-center only.
- **GitHub README** — matches the current viewer look (default). No running
  chrome by default.

Theme selection lives in `View → Theme` submenu and in the preview
toolbar. They are the same setting.

### Pagination rules

Smart defaults:

- `break-after: avoid` on all headings — a heading never ends a page
  without at least its first paragraph
- `break-inside: avoid` on `img`, `svg`, `figure`, `pre`, `table`
- Oversized image/diagram → scale proportionally to fit one page
- Oversized code block or table (too tall even scaled) → allowed to split
  as a last resort; clean sections stay whole

Optional: `break-before: page` on `h1` when "Start new page at H1" is on.

## Architecture

```
User action (⌘P / Export… / preview open)
          │
          ▼
PDFPreviewWindowController
          │
          ▼
PDFExporter.exportPDF(markdown, options) async throws -> Data
          │
          ├── MarkdownRenderer.renderFull(...)          (existing, shared)
          │
          ├── Hidden WKWebView sized to page dimensions
          │   • baseURL = bundle resources
          │   • inject pdf-overlay.css
          │   • inject @page rule from options
          │
          ├── DiagramRenderCoordinator awaits Mermaid / PlantUML completion
          │   (10s timeout fallback)
          │
          └── WKWebView.createPDF(configuration:) → Data
                    │
                    ▼
    PDFView + PDFThumbnailView (preview)
    NSPrintOperation (Print…)
    FileManager save panel (Export…)
```

The preview IS the output — a real PDF generated by the same pipeline
used for the final file. There is no secondary preview renderer.

## New components

### `MarkdownViewerKit`

- **`PDFExporter`** — orchestrates generation.
  `func exportPDF(markdown: String, options: PDFExportOptions) async throws -> Data`
- **`PDFExportOptions`** — value type:
  `theme`, `paperSize`, `orientation`, `startNewPageAtH1`, `showHeader`,
  `showFooter`, `headerText`.
- **`PDFTheme`** — `enum { .technical, .appleDocs, .gitHub }` with
  per-theme default chrome settings and CSS asset filename.
- **`DiagramRenderCoordinator`** — factored out of `WebContentView`; waits
  for Mermaid + PlantUML render completion, with a 10s timeout.

### `MarkdownViewer` (app)

- **`PDFPreviewWindowController`** — non-modal window described above.
  Owns a `FileWatcher`; regenerates PDF on file change and preserves
  current page index + zoom in `PDFView`.
- **Menu edits** in `AppDelegate`:
  - `File → Export as PDF…`
  - `View → Theme →` { Technical paper, Apple Documentation, GitHub }
  - `⌘P` routed through `PDFPreviewWindowController`
- **`ViewerWindowController`** edits — apply active theme to the content
  view; listen for theme changes.

### Resources

```
Resources/
  pdf-overlay.css            ← paged-media only: @page, break rules, white bg
  themes/
    technical.css
    appledocs.css
    github.css               ← default, mirrors current look
```

## Error handling

- **Diagram render timeout (10s):** generate the PDF anyway, badge the
  preview with a warning.
- **`createPDF` failure:** non-blocking sheet on the preview window with
  the `NSError` description.
- **Persistent oversized content:** let it clip on one page rather than
  loop or crash.
- **Missing bundle resource:** fall back to inline CSS; log once.

## Testing

`Tests/MarkdownViewerKitTests/PDFExporterTests.swift`, fixtures in
`Tests/MarkdownViewerKitTests/Fixtures/pdf/`:

- **Simple fixture** — assert page count == 1, extract text, compare to
  expected string.
- **Section-break fixture** (two `# H1` blocks) — with
  `startNewPageAtH1: true`, assert 2 pages.
- **Image-near-break fixture** — image positioned so naive pagination
  would split it. Assert its bounding box stays within a single page's
  bounds.
- **Large code block fixture** — assert it *does* split across pages
  (confirms the fallback).
- **Theme smoke tests** — render one fixture under each theme, assert
  non-empty PDFs with different byte sizes.

Preview window gets a manual QA checklist in this spec rather than UI
tests: live reload, theme switch, paper size swap, orientation swap,
warning badge appears on a page with a huge image.

## Out of scope

- Custom user-defined themes (future work; shipped themes cover the
  space for now)
- Cover page, table of contents generation, footnotes — the rendered
  markdown is what it is
- Batch export of multiple files (one-file-per-PDF for this iteration)
- Printing from the CLI `mdview` tool
